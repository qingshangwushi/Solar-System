## 二、修复详情

### 2.1 E-43：切比雪夫多项式发散（P0 阻断）

**文件**：`tools/ephemeris-pipeline/build_ephemeris.py`

**根因**：
- `_sample_times(period_days, n=64)` 完全忽略 `period_days`，在 200 年范围内仅采样 64 点
- 对地球（周期 365 天）：每 1142 天采一点 ≈ 3 个轨道周期，严重欠采样
- `n_segments_per_body = 4`（固定 4 段），每段 50 年，7 阶切比雪夫无法表示 ~50 个轨道振荡
- 结果：地球 J2000 位置 3.74 AU（应为 ~1 AU），段边界跳变 ~10 亿 km

**修复**：
1. `_sample_times`：自适应采样，每轨道周期 ≥16 点，最大间隔 30 天
   - 太阳：n=8（静止天体）
   - 地球：~4870 样本（原 64）
   - 月球：~42690 样本（原 64）
   - 总样本数：76,663（原 640）

2. `fit_chebyshev`：自适应分段，每段 ≥16 样本，段长 ≤365 天
   - 太阳：201 段（原 4）
   - 地球：201 段（原 4）
   - 月球：2673 段（原 4）
   - 总段数：5,235（原 40）

**验证结果**：
| 天体 | 修复前 | 修复后 | 期望值 | 误差 |
|------|--------|--------|--------|------|
| 地球 J2000 距离 | 3.74 AU | 1.018 AU | ~1.0 AU | 1.8% |
| 月球-地球距离 | N/A | 416,851 km | ~384,400 km | 8.4% |
| 太阳位置 | N/A | (0,0,0) | (0,0,0) | 0% |

> 注：误差来源于 Keplerian 简化模型（P2 精度），未含行星摄动。设计文档 13.4 允许 P2 级计算模型存在有限误差，但不得出现伪精度或发散。

### 2.2 E-44：NAIF ID 文件名别名缺失

**文件**：`tools/ephemeris-pipeline/build_ephemeris.py`

**根因**：构建管线仅写出简化 ID 文件名（`ephemeris-0.bin`, `ephemeris-3.bin`），但运行时编排器按 NAIF ID 索引（`ephemeris-10.bin`, `ephemeris-399.bin`）。

**修复**：
- 新增 `SIMPLE_ID_TO_NAIF` 映射表
- `main()` 写出步骤自动复制 NAIF ID 文件名别名

```python
SIMPLE_ID_TO_NAIF = {
    0: 10, 1: 199, 2: 299, 3: 399, 4: 499,
    5: 599, 6: 699, 7: 799, 8: 899, 301: 301,
}
```

**验证**：`release/data/` 同时包含简化 ID 和 NAIF ID 文件（共 22 个 .bin 文件）。

### 2.3 E-45：SSPH 解析器段数上限过低

**文件**：`packages/astro-core-api/src/ssph-parser.ts`

**根因**：解析器硬编码 `segmentCount > 1024` 上限，月球 2673 段超出限制导致注册失败。

**修复**：
```typescript
// 上限 65536：覆盖短周期天体在 200 年范围内的分段需求
// （例如月球 27 天周期 × 200 年 ≈ 2670 段），同时仍能拦截损坏文件。
if (segmentCount > 65536) {
  throw new SsphParseError(`SSPH 段数异常：${segmentCount}（上限 65536）`);
}
```

**验证**：新增测试用例 `解析真实 ephemeris-301.bin（Moon，短周期高分段数）`，验证 2673 段月球星历解析成功。
