## 四、新发现问题（7 项）

### N-01：renderer-core 地形相关导出不完整

**严重度**：P2
**描述**：`terrain.ts` 中定义的 `SurfaceCameraImpl`、`IrregularBodyRendererImpl`、`calculateScreenSpaceError`、`ElevationData` 等未从 `renderer-core/src/index.ts` 导出，导致外部包（如 terrain-engine）无法使用这些已修复的实现。
**证据**：`packages/renderer-core/src/index.ts:281-282`
**影响**：terrain-engine 只能自己实现低质量版本，已修复的代码无法复用

---

### N-02：ExtendedSpaceEnvironment.render() 不接收 renderer 参数

**严重度**：P1
**描述**：`ExtendedSpaceEnvironment.render()` 接口签名不接收 `renderer` 参数，子模块的 `render()` 被无参调用。而子模块只有在传入 renderer 时才会执行真实 GPU 绘制（drawPointList），无参时仅做 CPU 端预计算和统计更新。
**证据**：`packages/renderer-core/src/extended-space.ts:813-835`
**影响**：扩展空间的 7 类粒子效果（小行星带、柯伊伯带、奥尔特云、太阳风、磁层、极光、星空）在实际渲染管线中不会产生任何 GPU 绘制

### N-03：stellarBackground.update() 相机位置硬编码

## 六、核心阻塞链分析

当前项目无法运行出任何天体画面，其阻塞链路如下：

```
WASM 未构建 (E-41)
    ↓
Worker 无法加载天文内核 (R-03)
    ↓
app-orchestrator 为空 (E-25) ──→ 无启动编排
    ↓
App.tsx 用 setTimeout 模拟 (E-35)
    ↓
SceneViewport 仅占位 emoji (E-35)
    ↓
无 canvas → 无 renderer → 无 draw call
    ↓
R-02 数据流断裂：即使 renderer 存在，也无人调用 subscribeSnapshot → BodyRenderer.update()
```

**最短路径打通方案**（先让画面出来再谈质量）：
1. 构建 WASM → 生成 pkg/ 目录
2. 实现最小 app-orchestrator（diagnostics → Worker init → renderer 创建 → body renderer 注册）
3. SceneViewport 创建 canvas 并挂载 renderer
4. 建立 snapshot → body.update() 的数据流
5. 跑通"太阳 + 地球 + 月球"的首帧渲染
