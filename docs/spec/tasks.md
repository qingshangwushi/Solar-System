# 开发任务清单 (tasks.md)

> 每个任务均含【产出】与【验证】。验证必须实际执行并通过后才标记完成。
> 阶段顺序受设计文档附录 C 技术门禁约束，未通过门禁不进入下一阶段。
> 任务编号：`P<阶段>-<序号>`。阶段 0 为本次执行重点（完整可运行底座）。

---

## 阶段 0：技术风险验证（本次执行重点）

### P0-1 Monorepo 与工具链初始化

- **产出**：
  - pnpm workspace（`apps/`、`packages/`）+ Cargo workspace（`crates/`）；

- **验证**：`pnpm install` 成功；`pnpm -r typecheck` 通过；`cargo check` 通过。

### P0-2 目录骨架与包占位

- **产出**：按设计文档第 41 节创建全部目录与空包占位（`packages/*`、`crates/*`、`tools/*`、`data-src/*`、`assets-src/*`、`release/*`、`tests/*`、`docs/`）；每个 TS 包含 `package.json`、`tsconfig.json`、`src/index.ts`；每个 Rust crate 含 `Cargo.toml`、`src/lib.rs`。
- **验证**：`pnpm -r typecheck` 与 `cargo check --workspace` 全通过；目录树与设计文档一致。

### P0-3 Schemas 包：版本化数据 Schema

- **产出**：`packages/schemas` 内定义 JSON Schema（带版本号）：

- **验证**：`pnpm -r typecheck` 通过；用样本 JSON 实例通过 ajv 校验。

### P0-4 astro-core-api：Worker 消息协议

- **产出**：`packages/astro-core-api` 定义：
  - `SimulationClock`、`EphemerisProvider`、`CelestialStateSnapshot`、`RenderBridge`、`ResourceManager`、`EventEngine`、`TourPlayer` 接口（按设计文档 42 节）；

### P0-5 Rust 时间系统（crates/time-system）

- **产出**：
  - JD/UTC/TAI/TT/TDB 表示与转换；
  - 内置闰秒表（数据包可替换）；当前日期前正式闰秒，未来"最后已知 + 预测 Delta T"并返回不确定性标记；
  - 时间倍率与边界（1900-01-01 … 2100-12-31，超范围返回 `OUT_OF_RANGE`）；
  - 连续时间尺度内部计算，避免 UTC 跳秒导致轨道不连续。
- **验证**：`cargo test -p time-system`：UTC/TAI/TT/TDB 转换已知样本、闰秒边界、超范围降级；与 IERS 已知转换点对照。

### P0-6 Rust 坐标系统（crates/coordinate-system）

- **产出**：
  - 双精度向量/矩阵/四元数；
  - 参考系枚举（SSBI/日心惯性/行星质心/天体固连/地表 ENU/观察者相对）；
  - 参考系转换；禁止跨系直接相加的编译期/运行期守护；
  - 高低位拆分（f64→双 f32）工具，供渲染层使用。
- **验证**：`cargo test -p coordinate-system`：已知坐标转换、高低位往返误差 < 1e-3（渲染精度）。

### P0-7 星历运行时（crates/ephemeris-runtime + 构建管线骨架）

- **产出**：
  - 分段切比雪夫系数插值（位置/速度）；
  - 按 (bodyId, TDB, 参考系) 定位系数块；超范围走 fallback；
  - `tools/ephemeris-pipeline` Python 骨架：SPK 读取 → 1900—2100 裁剪 → 切比雪夫拟合 → 误差分析 → 紧凑二进制 + 索引 + 校验报告（设计文档 14.2）；
  - 内置一份地月简化星历样本（用于离线冒烟；明确标注精度等级与来源）。
- **验证**：`cargo test -p ephemeris-runtime`：切比雪夫插值与解析解对照误差达标；超范围返回 fallback；样本星历可加载并求值。

### P0-8 astro-core 内核整合（crates/astro-core）
